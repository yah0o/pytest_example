Dummy Server Readme

Endpoints

  • GET /readme
  • GET /routes
  • GET /registry
  • GET /logs/:registry_key
  • POST /unregister
  • POST /register
  • POST /attach/:registry_key

    • Description

    GET /readme

    Brings up documentation.

    GET /routes

    Returns a list of registered routes and methods

    GET /registry

    Returns all of the registered routes and the corresponding data

    GET /logs/:registry_key

    Returns the logs for a certain call

    POST /unregister

    Unregisters a call.

    Request

        {
        method: string,     (required)
        path: string        (required)
    }

    method - http method "Get" or "Post"

    path - the path to unregister

    Response

        {
        is_registered: boolean,
        method: method,
        path: string
    }

    POST /register

    registers a call.

    Request

        {
        method: string,     (required)
        path: string        (required)
        headers: object     (optional)
        status_code: object (optional)
        content: object     (optional)
    }

    method - http method "Get" or "Post"

    path - the path to unregister

    headers - headers the call will respond with when path is called

    content - content the call will respond with when path is called

    status_code - status the call will respond with when the path is called

    Response

        {
        is_registered: boolean,
        content: object,
        log: array,
        method: string,
        headers: object,
        status_code: int,
        path: string,
        registry_key: string
    }

    POST /attach

    attach a file that will be returned when the call is called.

    Call must be registered before using the registered call
